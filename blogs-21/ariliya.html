CSS Programming Language


Ariliya Sarkar

Introduction to CSS

CSS is also known as: Cascading Style Sheet.

  1. HTML defines structure and content of a web page. (HTML- Hypertext Markup Language)
  2. A cascading style sheet is a web document that permits to change the appearance of the HTML.
  3. CSS permits to change the size, style, font, and colour of text, It also permits to change the margins and padding, background colours and border styles.

Advantages of CSS

CSS is consistent.
The best thing about CSS is that is allows us to make global style changes that :
Every part of a certain element is effected throughout the website so that we don’t have to make these changes at individual page. This saves time when it comes to redesigning.

Working of CSS:

THE CASCADE A very important part of CSS is the Cascading part.>

  1. The definitions are read by the browser, of the styles from top to bottom in a style sheet. This means that the defined style lower in the style sheet will override any previous styles defined earlier in the style sheet.
  2. We can also have a style sheet over-ride another style sheet. This is how we are able to over-ride predefined styles, as a custom style sheet is usually the last one read by the browser.

How is CSS applied?

In HTML, CSS is applied, elements in a web page by declaring specific styles for each element. A style declaration looks like: 

            
                selector {
                    property: value;
                }
Selector :
The selector is the small part or piece of content that we want to target and style. It is either an element of HTML or a CLASS name. When an HTML element is defined, we use the tag name without the opening and closing bracket. For example, 
                    
                        <p> (or paragraph tag) would simply be: p
                        A Class Name always begins with a dot. For example,
                        .big-font
                        We’ll get more into classes in a bit.
Property :
Properties and their respective values always live in braces: 
            
                p {
                }
Within CSS properties are predefined terms that all web browsers understand. They are things like: 
            
                font-family, font-size, color, etc.

                    p {
                        font-knight: 
                        font-size: 
                        color: 
                    }
Mandatory: Property is always followed by colon (:)
Value : The value is the style, indeed a particular style or variable you choose to assign to a property. For example: 
            
                p {
                    font-knight:Calibri light;
                    font-size: 20px;
                    color: grey;
                }
A value is always followed by a semi-colon (;)
So the example above tells the browser that we want all of our page titles (or any element surrounded by an 

 tag) to be displayed in Arial font at 16 pixels in size and in the color gray. Pretty easy, right? Let’s do another one. 

    
        a {
            color: blue;
            font-weight: bold;
            text-decoration: none;
        }

Syntax Errors

Sometimes after editing the style sheet, we might get an error or find that a particular style definition is not working when we refresh our browser window. Oftentimes, these are caused due to syntax errors.

  • Browsers might be very particular when it comes to proper syntax and one little mistake can throw everything off.
